Q3 model answer

library(tidyverse)
library(readxl)

install.packages("readxl")   # run once in the Console, not in the script

Q4 model answer

library(readxl)   # this line was missing

tree_df <- read_excel("data_raw/tree_data.xlsx")

install.packages() downloads the package to the computer once. library() must be called every new session to activate it. Without the library() line, R does not know read_excel() exists.

Q7 model answer

side <- c("sunny", "shady")   # quotes are required around character values

Without quotes, R treats sunny and shady as object names and throws object 'sunny' not found.

Q9 model answer

ggplot(tree_df, aes(x = side, y = weight_g)) +

Q10 model answer — three errors

ggplot(tree_df, aes(x = side, y = weight_g)) +   # Fix 1: + must end this line
  geom_boxplot() +
  labs(x = "Side of tree",                        # Fix 2: missing comma after first argument
       y = "Leaf weight (g)")

Fix 1: the + that connects ggplot() to geom_boxplot() was missing at the end of the first line — it must sit at the end of a line, never the start of the next.

Fix 2: inside labs() a comma is missing after x = "Side of tree" — each argument must be separated by a comma.

Fix 3 (implicit): the original had + at the start of the geom_boxplot() line, which causes an error — the + belongs at the end of the preceding line.
